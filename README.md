## Governance patterns I care about

- Clear prototype scope boundaries
- Human approval before downstream action
- Evidence records for decisions and exceptions
- Mock adapters before real integrations
- Cost gates before cloud work
- Release checks before push or deployment
- No customer data, credentials, or internal system details in public artifacts

## What this profile is

This profile is a career proof-of-work surface for enterprise AI delivery and governance thinking.

It is meant to help reviewers inspect how I structure AI delivery work: planning, controls, evidence, review gates, and safe iteration.

## What this profile is not

- Not a production banking implementation
- Not a claim of owning a live enterprise AI platform
- Not an AI model training or fine-tuning portfolio
- Not a place for real customer data, internal systems, credentials, or private company information

## Delivery principles

- Small, reviewable changes
- Docs before code when risk is unclear
- Mock before real API
- Human decides; AI does not self-approve
- Stop on secret risk, cost risk, scope creep, or unclear repo path
